# Task Briefs

Use `ai/tasks/*.md` for feature briefs that need to persist across sessions.

## How To Use

1. Create or update a task brief when the work is multi-step, uncertain, or likely to span sessions.
2. Keep briefs compact and decision-oriented.
3. Link local docs, issues, and relevant implementation files when they materially help.
4. Update the brief only when scope, requirements, or accepted tradeoffs change.

## Active Task Brief

- Read the task brief that matches the work in progress.
- If no brief exists yet, create one from [`ai/tasks/_template.md`](./_template.md).

## Naming

- Prefer durable feature-oriented names such as `agent-widget-followups.md` or `contact-form-v2.md`.
- Prefix with a date only when concurrent briefs need disambiguation.
