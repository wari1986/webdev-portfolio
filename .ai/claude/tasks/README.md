# Task Briefs

Use `.ai/claude/tasks/*.md` for work that spans sessions, has ambiguous scope, or needs durable acceptance criteria.

## How To Use

1. Create or update the active brief before implementation when the task is multi-step or uncertain.
2. Keep each brief compact and specific to a real feature or fix.
3. Capture only the information needed to continue the work cleanly in a later session.
4. Update the brief when scope, decisions, or validation criteria materially change.

## Active Task Brief

- Read the brief that matches the current work.
- If no brief exists, create one from [`.ai/claude/tasks/_template.md`](./_template.md).

## Naming

- Prefer durable, feature-oriented names such as `portfolio-agent-polish.md` or `mobile-canvas-regression.md`.
