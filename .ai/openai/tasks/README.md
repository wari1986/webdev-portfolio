# Task Briefs

Use `.ai/openai/tasks/*.md` for work that needs durable scope, decisions, or acceptance criteria across sessions.

## How To Use

1. Create or update the active task brief before implementation when the work is multi-step or uncertain.
2. Keep each brief compact and decision-oriented.
3. Capture only information that will help continue the work later.
4. Update the brief when scope, decisions, or validation meaningfully change.

## Active Task Brief

- Read the brief that matches the current work.
- If none exists, create one from [`.ai/openai/tasks/_template.md`](./_template.md).

## Naming

- Prefer durable names such as `portfolio-agent-polish.md`, `showcase-section.md`, or `mobile-canvas-fix.md`.
